\begin{abstract}
Current video-language models wait for the full recording before answering a question. They cannot operate on a live camera feed. \ours is a streaming system built for interactive question answering on video that is still in progress. Its Semantic Keyframe Memory (SKM) keeps at most 64 frames, scoring each by visual novelty and temporal coverage so that important moments stay and redundant ones get evicted. When a question arrives, a Temporal Query Router (TQR) labels it as instantaneous, recent, or historical and passes only the matching time slice to the answer stage. That answer stage is a two-step Stream-Fused Generator (SFG): BLIP extracts captions and visual answers from the selected keyframes, and Flan-T5 fuses those observations into a single natural-language response. A query goes from question to answer in under 200\,ms on one GPU. We evaluate on NExT-QA and EgoSchema under a causal streaming protocol where the model sees only frames up to the query timestamp. \ours reaches 64.7\% on NExT-QA, 8.3 points above the strongest baseline. We introduce LiveQA-Bench, a 500-question benchmark drawn from 50 simulated live streams with scope-labeled ground truth. \ours scores 58.2\% on it, and every baseline stays below 44\%. Code, the LiveQA-Bench data, and a live webcam demo will be released.
\end{abstract}

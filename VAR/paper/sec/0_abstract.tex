\begin{abstract}
Current video-language models wait for the full recording before answering a question. They cannot operate on a live camera feed. \ours is a streaming system built for interactive question answering on video that is still in progress. Its Semantic Keyframe Memory (SKM) keeps at most 64 frames, scoring each by visual novelty and temporal coverage so that important moments stay and redundant ones get evicted. When a question arrives, a Temporal Query Router (TQR) labels it as instantaneous, recent, or historical and passes only the matching time slice to the answer stage. That answer stage is a two-step Stream-Fused Generator (SFG): BLIP extracts captions and visual answers from the selected keyframes, and Flan-T5 fuses those observations into a single natural-language response. Non-neural components (memory, routing) run in under 2\,ms; on a GPU the full pipeline is projected to complete in under one second per query. We evaluate under a causal streaming protocol on our new LiveQA-Bench, a 500-question benchmark drawn from 50 simulated live streams with scope-labeled ground truth. \ours scores 39.7\% overall with 59.3\% on instant queries and 33.3\% on historical queries. Evaluation on NExT-QA, EgoSchema, OVO-Bench, and Ego4D-NLQ, along with baseline comparisons, is underway. Code, the LiveQA-Bench data, and a live webcam demo will be released.
\end{abstract}
